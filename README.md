# 防窥屏程序 (Privacy Shield)

一个基于面部识别的智能防窥屏程序，当检测到陌生人时自动黑屏，保护你的隐私安全。


## 📋 项目简介

防窥屏程序是一款专为保护个人隐私设计的工具，通过摄像头实时检测用户面部，区分主人和陌生人。当检测到陌生人时，程序会自动显示黑屏，防止他人窥视你的屏幕内容。

### 核心功能

- 🎯 **面部识别**：注册主人面部信息，支持多个角度的特征采集
- 🔍 **实时检测**：默认显示摄像头实时检测画面，带有检测框
- ⚠️ **陌生人检测**：当检测到陌生人时自动黑屏
- 📋 **系统托盘**：通过系统托盘图标管理程序
- 🛠️ **配置向导**：首次启动时引导用户配置
- ⚡ **热键支持**：使用 `Ctrl+Alt+P` 快速启用/禁用保护
- 📸 **截图功能**：检测到陌生人时自动保存截图

## 🎯 使用指南

### 首次启动

1. **配置向导**：首次启动时会自动弹出配置向导
2. **注册主人信息**：按照提示面对摄像头，做出不同姿势完成注册
3. **摄像头设置**：程序会使用默认摄像头
4. **完成配置**：点击"完成"按钮，程序开始保护

### 日常使用

- **启用/禁用保护**：通过系统托盘图标 → 启用保护/禁用保护
- **查看实时检测**：默认显示摄像头实时检测画面，带有检测框
- **解除黑屏**：当检测到陌生人并黑屏时，点击鼠标左键三下即可解除
- **热键操作**：使用 `Ctrl+Alt+P` 快速切换保护状态

### 系统托盘菜单

- **启用保护**：开始隐私保护
- **禁用保护**：暂时停止保护
- **注册主人**：重新注册主人面部信息
- **配置向导**：重新运行配置向导
- **设置**：打开详细设置窗口
- **退出**：退出程序


## ⚙️ 配置说明

配置文件：`privacy_shield_config.json`

### 主要配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `sensitivity` | 0.45 | 识别灵敏度 (0.1-1.0)，值越高识别越严格 |
| `response_time` | 0.5 | 响应时间 (秒)，检测到陌生人后多久黑屏 |
| `show_camera` | true | 是否显示摄像头画面 |
| `camera_index` | 0 | 摄像头索引 |
| `startup` | false | 是否开机自启动 |
| `screenshot_path` | `C:\Users\mahaoxing\Pictures\pricay` | 截图保存路径 |

### 调整配置

1. 通过系统托盘 → 设置，打开设置窗口
2. 修改相应的配置项
3. 点击"保存"按钮应用更改

## 🔧 技术实现

### 面部识别原理

1. **面部检测**：使用OpenCV的Haar级联分类器检测人脸
2. **特征提取**：计算面部的直方图特征
3. **相似度比较**：使用相关系数比较面部特征
4. **自适应灵敏度**：根据环境光线自动调整识别灵敏度

### 核心模块

- **register_owner_wizard()**：注册主人面部信息
- **detection_loop()**：实时检测循环
- **compare_faces()**：比较面部特征
- **show_black_screen()**：显示黑屏
- **create_tray_icon()**：创建系统托盘图标


## 🛡️ 安全说明

- **隐私保护**：所有面部数据仅存储在本地，不会上传到任何服务器
- **数据安全**：主人信息使用pickle格式存储在本地文件中
- **权限要求**：程序需要摄像头访问权限
- **资源占用**：实时显示摄像头画面会增加系统资源占用


## 📖 常见问题

### 问题：摄像头无法打开

**解决方案**：
- 检查摄像头连接是否正常
- 确保摄像头没有被其他程序占用
- 尝试重启计算机
- 检查是否有摄像头访问权限

### 问题：人脸识别失败

**解决方案**：
- 确保光线充足
- 面对摄像头保持正面
- 确保面部在摄像头视野内
- 尝试重新注册主人信息

### 问题：程序崩溃

**解决方案**：
- 查看 `privacy_shield.log` 文件了解错误信息
- 尝试重新安装依赖
- 检查Python版本是否兼容（推荐Python 3.7+）

### 问题：黑屏无法解除

**解决方案**：
- 连续点击鼠标左键三下
- 如果仍无法解除，尝试重启程序
